# HR Analytics and Database Project

## Overview

I developed this project to build an employee management database and analyze workforce data using SQL. The goal was to transform raw employee records stored in CSV files into a structured database and generate useful reports about employees, departments, salaries, and organizational relationships.

I designed the database structure, imported and organized employee data, and created SQL queries to answer common HR analytics questions.

## Project Objectives

- Build a relational database for managing employee information.
- Organize raw HR data into structured tables.
- Create relationships between employees, departments, managers, salaries, and job information.
- Use SQL queries to generate workforce insights and reports.

## Project Workflow

### 1. Database Design

I designed the database structure by creating an Entity Relationship Diagram (ERD) to define how employee data is connected across different tables.

The database includes information related to:

- Employees
- Departments
- Department managers
- Salaries
- Job titles
- Employee department assignments

### 2. Data Engineering

I created database tables from six CSV files and organized the data into a relational database structure.

My work included:

- Creating table schemas for each dataset.
- Defining primary keys and relationships between tables.
- Importing employee records from CSV files.
- Organizing raw data into a format suitable for analysis.

### 3. Data Analysis

I wrote SQL queries to analyze employee and organizational data.

The analysis included:

- Retrieving employee details such as employee number, name, gender, and salary.
- Finding employees who were hired during a specific time period.
- Identifying department managers and their assigned departments.
- Listing employees along with their department information.
- Searching employee records based on specific name patterns.
- Generating lists of employees working in specific departments.
- Comparing employee distribution across different departments.
- Analyzing the frequency of employee last names.

## Key Tasks Completed

- Designed an ERD for an HR database system.
- Created SQL table structures from multiple CSV datasets.
- Imported and organized employee data into a relational database.
- Developed queries for employee, salary, and department reports.
- Built reports showing department structure and employee assignments.
- Combined information from multiple tables to answer HR-related questions.
- Validated SQL queries to ensure accurate results.

## Technologies Used

- SQL
- PostgreSQL / MySQL
- CSV Data Processing
- Entity Relationship Diagrams (ERD)
- Relational Database Design

## Project Outcome

This project demonstrates how raw employee data can be transformed into a structured HR database and used to generate meaningful workforce reports. The final database supports employee lookup, organizational analysis, and HR reporting through SQL queries.
